## 1. Mathematical model

Domain:
```text
Omega = (0,1)^2
Gamma_D = empty
Gamma_I = boundary(Omega)
```

Global Helmholtz problem:
```text
-Delta u - (k^2 + i*eps_prob) u = f              in Omega
du/dn - i*eta_prob*u = g                         on boundary(Omega)
```

The weak form uses
```text
a_eps(u,v) = int_Omega grad(u).grad(conj(v))
             - (k^2 + i*eps_prob) int_Omega u*conj(v)
             - i*eta_prob int_boundary u*conj(v)
```

Global matrix:
```text
A_prob = S - (k^2 + i*eps_prob)*M - i*eta_prob*N_boundary
```
where `S` is stiffness, `M` is the volume mass matrix, and `N_boundary` is the boundary mass matrix.

Default exact solution used for the right-hand side:
```text
d = (1/sqrt(2), 1/sqrt(2))
u_exact(x,y) = exp(i*k*(x*d[0] + y*d[1]))
```

Then
```text
f = -i*eps_prob*u_exact
g = i*(k*(d dot n) - eta_prob)*u_exact
```
because `Delta u_exact = -k^2*u_exact`.

Use `eta_prob = k` and `eta_prec = k` as the first implementation choice. If the counts do not match, run a second branch with `eta = sqrt(k^2 + i*eps)` for both global and local problems. The paper permits both conventions in the analysis, while Section 4 does not restate which convention is used in the computations.

## 2. Finite-element discretization

Use conforming continuous piecewise-linear finite elements (`P1`) on a uniform triangular mesh.

Recommended explicit mesh construction:
1. Start from a uniform square grid with `Nfine` intervals per coordinate.
2. Split each square into two triangles using the same diagonal throughout the grid.
3. Include boundary nodes as unknowns because `Gamma_D` is empty.

Element matrices:
```text
local stiffness: S_e[a,b] = area * grad(phi_a).grad(phi_b)
local mass:      M_e = area/12 * [[2,1,1],[1,2,1],[1,1,2]]
boundary mass on boundary edge of length L:
                N_e = L/6 * [[2,1],[1,2]]
```

Fine mesh size convention:
```text
Ntarget = round(k^(3/2))
```

Because local subdomains are unions of fine triangles and are induced by a coarse grid, force `Nfine` to be divisible by the coarse parameter `Mcoarse`.

Suggested rounding:
```python
def choose_M_from_alpha(k, alpha, mode="round"):
    val = k**alpha
    if mode == "round":
        return max(1, int(round(val)))
    if mode == "ceil":
        return max(1, int(math.ceil(val)))
    if mode == "floor":
        return max(1, int(math.floor(val)))
    raise ValueError(mode)

def choose_Nfine(k, Mcoarse):
    Ntarget = int(round(k**1.5))
    return Mcoarse * max(1, int(round(Ntarget / Mcoarse)))
```

Calibration advice:
- Try `Mcoarse = round(k**alpha)` first.
- If iteration counts are consistently too high/low, try `ceil(k**alpha)` and `floor(k**alpha)`.
- Keep a log of `k, alpha, Mcoarse, H=1/Mcoarse, Nfine, h=1/Nfine, n_dofs`.

The paper's notation says `h ~ k^(-3/2)` and `H ~ k^(-alpha)`, not exact equality. Thus the integer choices above are implementation conventions, not guaranteed author choices.

## 3. Overlapping subdomains and partition of unity

Given a uniform coarse mesh of `Mcoarse x Mcoarse` square elements, with coarse nodes
```text
x_{ell,m} = (ell/Mcoarse, m/Mcoarse),    ell,m = 0,...,Mcoarse.
```

Define one subdomain per coarse node:
```text
Omega_{ell,m} = interior of union of all coarse square elements touching x_{ell,m}.
```

Equivalent rectangular patch:
```text
x range: [max(0,(ell-1)/Mcoarse), min(1,(ell+1)/Mcoarse)]
y range: [max(0,(m-1)/Mcoarse),   min(1,(m+1)/Mcoarse)]
```
with the obvious truncation at the boundary.

Number of subdomains:
```text
Nsub = (Mcoarse + 1)^2
```

Partition of unity:
Use the tensor-product bilinear coarse-grid nodal basis function
```text
chi_{ell,m}(x,y) = max(1 - abs(Mcoarse*x - ell), 0)
                  * max(1 - abs(Mcoarse*y - m), 0)
```
evaluated at fine-grid nodes. These functions sum to one on `Omega`.

For each subdomain:
- `local_nodes`: all fine nodes belonging to the patch.
- `chi_local`: values of `chi_{ell,m}` at those nodes.
- Restriction with weights:
  ```text
  R_l v = chi_local * v[local_nodes]
  ```
- Prolongation:
  ```text
  R_l^T w adds chi_local * w into global entries local_nodes.
  ```

This is the SORAS/OBDD-H form:
```text
B_prec^{-1} = sum_l R_l^T * inv(A_prec,l) * R_l
```

## 4. Local subproblem matrices

For the standard impedance subdomain experiments, assemble
```text
A_prec,l = S_l - (k^2 + i*eps_prec)*M_l - i*eta_prec*N_l_boundary
```
on the local patch `Omega_l`.

Here `N_l_boundary` is the boundary mass matrix over the entire boundary of the local patch. Since `Gamma_D` is empty globally, all local patch boundaries carry the impedance term in the impedance version.

For Experiment 5, replace local impedance conditions with homogeneous Dirichlet conditions on subdomain boundaries.

Implementation choice for Experiment 5:
- Primary version: local unknowns are only the strictly interior fine nodes of the local patch; nodes on `boundary(Omega_l)` are eliminated as zero Dirichlet nodes.
- Equivalent algebraic version: assemble the local matrix on the patch and take the submatrix for interior local nodes.
- Restriction/prolongation in this case acts only on those interior local nodes.

If this version gives a systematic mismatch with Table 4, test a secondary version in which the local boundary mass term is removed but local boundary DOFs are kept unconstrained. The paper's wording can be read as "remove the local impedance boundary integral," whereas the table caption says "homogeneous Dirichlet condition on subdomain boundaries." Log which convention is used.

## 5. GMRES setup

Use left-preconditioned GMRES:
```text
C x = B_prec^{-1} * A_prob * x
b_tilde = B_prec^{-1} * F
solve C x = b_tilde
```

Use standard unrestarted GMRES with Euclidean residual minimization. Stop when
```text
||r_j||_2 / ||r_0||_2 <= 1e-6
```
where `r_j = b_tilde - C*x_j` for the left-preconditioned system.

Initial guess:
- Experiments 1, 2, 3, and 5: random real vector uniformly distributed in `[0,1]^n`.
- Experiment 4: compare the random-start and zero-start variants exactly as listed below.

Because the random seed is not specified, fix a seed for reproducibility, e.g.
```python
rng = numpy.random.default_rng(20240528)
x0 = rng.uniform(0.0, 1.0, size=n_dofs).astype(complex)
```
Run several seeds if exact counts fluctuate; report median and seed-specific counts.

No restart:
- In PETSc, use `KSPGMRESSetRestart(ksp, n_dofs)` or a very large restart.
- In SciPy, set `restart` larger than the expected iteration count; for large runs, PETSc is preferred.

## 6. Solver and implementation strategy

For small tests, pure SciPy is acceptable:
- sparse CSR matrices for global and local matrices;
- `scipy.sparse.linalg.splu` for each local matrix;
- `scipy.sparse.linalg.gmres` with a `LinearOperator`.

For the full tables, use PETSc/petsc4py:
- Matrix assembly in parallel or domain-wise.
- Local subdomain solvers with LU factorization via MUMPS/SuperLU_DIST or sparse direct solver.
- Preconditioner apply: parallel local solves, then weighted accumulation.
- The largest cases have millions of DOFs; a serial Python implementation is mainly for debugging and reduced-size reproduction.

Pseudo-code for one run:
```python
def run_case(k, eps_prob, eps_prec, alpha=None, M_fixed=None,
             local_bc="impedance", x0_type="random", seed=20240528):
    if M_fixed is None:
        Mcoarse = choose_M_from_alpha(k, alpha, mode="round")
    else:
        Mcoarse = M_fixed

    Nfine = choose_Nfine(k, Mcoarse)
    mesh = build_unit_square_tri_mesh(Nfine)

    A_prob, F = assemble_global_helmholtz(mesh, k, eps_prob, eta_prob=k,
                                          exact_plane_wave=True)

    subdomains = build_vertex_patch_subdomains(mesh, Mcoarse)
    factors = []
    for sd in subdomains:
        A_loc = assemble_local_matrix(mesh, sd, k, eps_prec, eta_prec=k,
                                      local_bc=local_bc)
        factors.append(factorize(A_loc))

    def apply_Binv(r):
        z = zeros_like(r)
        for sd, fac in zip(subdomains, factors):
            rloc = sd.chi * r[sd.nodes_for_solver]
            wloc = fac.solve(rloc)
            z[sd.nodes_for_solver] += sd.chi * wloc
        return z

    def C_action(v):
        return apply_Binv(A_prob @ v)

    btilde = apply_Binv(F)

    x0 = make_initial_guess(n=A_prob.shape[0], kind=x0_type, seed=seed)
    its = gmres_count_iterations(C_action, btilde, x0=x0, rtol=1e-6,
                                 norm="euclidean", restart=None)
    return its
```

For Experiment 5, if using strict Dirichlet local solves:
```text
sd.nodes_for_solver = local patch nodes excluding patch-boundary nodes
sd.chi = chi values restricted to sd.nodes_for_solver
```

## 8. Verification checklist

A run is likely correct if all of the following hold:

1. Table 1(a) counts stay nearly bounded in `k` for every fixed `alpha`.
2. Table 1(b) has larger counts than Table 1(a), especially for larger `alpha`.
3. Table 2 shows similar performance for `eps_prec=k` and `eps_prec=0`.
4. Table 3 shows larger iteration counts for larger fixed `M`; random-start counts are lower than zero-start counts.
5. Table 4 is much worse than Table 2/Experiment 3, especially for large `alpha` and large `k`.
6. For fixed `k`, increasing `alpha` generally increases the iteration count because subdomains are smaller.
7. Logs include the actual `Mcoarse`, `Nfine`, `n_dofs`, `eta` convention, random seed, local boundary condition convention, GMRES residual definition, and software versions.

## 9. Common pitfalls

- Do not impose global Dirichlet boundary conditions; the experiments use `Gamma_D = empty`.
- Do not omit the physical boundary impedance term in the global matrix.
- For impedance local solves, include the local impedance boundary term on artificial subdomain boundaries.
- Use weighted restriction/prolongation via the partition of unity. An unweighted additive Schwarz preconditioner is not the same experiment.
- Do not use restarted GMRES with a small restart length; this changes iteration counts.
- Distinguish `eps_prob` from `eps_prec`.
- Ensure local subdomain patches align with the fine mesh.
- Log whether `H ~ k^(-alpha)` uses `round`, `ceil`, or `floor` for `Mcoarse`.
- For the exact plane-wave right-hand side with `eps_prob=0`, the volume source is zero, but the boundary load is generally nonzero.
